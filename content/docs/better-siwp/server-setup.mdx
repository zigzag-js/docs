## Prerequisites

- A Better Auth instance ([setup guide](https://www.better-auth.com/docs/installation))
- A database adapter configured with Better Auth (Drizzle, Prisma, etc.)

## Installation

```bash
npm i @zig-zag/better-siwp better-auth
```

## Basic Setup

Add the `siwp()` plugin to your Better Auth config:

```ts title="lib/auth.ts"
import { betterAuth } from 'better-auth';
import { siwp } from '@zig-zag/better-siwp';

export const auth = betterAuth({
  database: {
    // your database adapter
  },
  plugins: [siwp()],
});
```

## Configuration Options

The `siwp()` plugin accepts an optional config object:

```ts
siwp({
  // Custom statement shown in the signing prompt
  statement: 'Sign in to MyApp',

  // Domain for the SIWS message (auto-detected by default)
  domain: 'myapp.com',
})
```

### Options

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `statement` | `string` | `"Sign in with Polkadot"` | Message shown in the wallet signing prompt |
| `domain` | `string` | Auto-detected from request | Domain for the SIWS message |

## API Routes

The plugin adds these endpoints to your Better Auth instance:

| Endpoint | Method | Description |
| --- | --- | --- |
| `/api/auth/siwp/nonce` | `GET` | Generate a nonce for signing |
| `/api/auth/siwp/verify` | `POST` | Verify signature and create session |

## Next.js App Router

```ts title="app/api/auth/[...all]/route.ts"
import { auth } from '@/lib/auth';
import { toNextJsHandler } from 'better-auth/next-js';

export const { GET, POST } = toNextJsHandler(auth);
```
